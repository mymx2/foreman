# 证据来源优先级

进行任何技术判断时，按以下证据层级执行。当高层级来源**可用且相关**时，优先使用高层级，禁止用低层级替代。某层级不可用时（工具缺失、无网络、目录不存在），跳过并降级到下一个可用层级，同时标注降级原因。

## 证据层级

| 层级 | 来源 | 适用场景 | Token 成本 |
|------|------|----------|------------|
| P0 | **工作区代码**（如文本搜索、符号解析、终端执行、数据库查询等） | 读取项目文件、运行构建和测试、查看数据库结构 | 低（本地读取） |
| P1 | **依赖源码文件**（如 IDEA-MCP、LSP 等） | 理解第三方库行为、API 内部实现、类型签名 | 低–中 |
| P2 | **文档查询工具**（如 Context7 MCP 等） | 框架、库、构建工具的官方文档。工具不可用时跳过 | 中 |
| P3 | **网络检索**（如 WebSearch/WebFetch 等） | 最新资讯、P2 未覆盖的内容、错误信息、发布说明。无网络时跳过 | 中–高 |
| P4 | **结构化记忆**（如 SearchMemory 等记忆工具） | 用户偏好、项目模式、跨项目经验。工具不可用时跳过 | 低 |
| P5 | **训练知识** | 所有外部来源均不可用时的最终降级，或安全豁免主题（见下文） | 零 |

### 核心约束

- **P0–P1**：必须用工具读取实际文件。禁止凭记忆引用文件内容。
- **P2–P3**：涉及 API 签名、版本特定行为、废弃状态或构建工具配置时，**若工具可用**必须使用。
- **P4**：作为补充上下文。行动前应用更高可用层级验证事实。
- **P5**：所有外部来源不可用时允许作为最终降级。用于非平凡判断时，标注 `(inferred: 未联网验证)`。若属于下文「禁止 P5」的领域，即使无外部工具也应明确告知用户无法验证，而非自信给出结论。

### 降级策略

当某层级不可用时，不阻塞任务，直接降级到下一个可用层级：

| 不可用情况 | 处理方式 |
|------------|----------|
| P1 依赖源码目录不存在 | 跳到 P2/P3 查文档，或读项目代码中的用法推断 |
| 文档查询工具不可用 | 跳到 P3 网络检索，或 P5 标注未验证 |
| 无网络访问 | 依赖 P0/P1 本地文件 + P4 记忆，P5 标注 `(offline)` |
| 记忆工具不可用 | 跳过 P4，直接 P5 |
| 所有外部来源均不可用 | P5 作为最终降级，但必须明确告知用户：哪些结论未经验证、风险范围、恢复验证的方法 |

## 依赖源码目录映射

需要 P1 检索时，按生态系统在本地查找源码。以下为各生态系统的**典型路径**，具体路径因项目配置而异——若典型路径不存在，在项目目录或全局缓存中搜索对应的源码目录：

| 生态系统 | 典型本地源码路径 | 说明 |
|----------|-------------|------|
| Gradle (Kotlin/Java) | `<project>/.gradle/gradle_module/<group>/<artifact>/<version>/` | 需项目配置 `unzipSourceJars` 任务；未配置时回退到 `~/.gradle/caches/` 下查找 `-sources.jar` |
| Maven (Java) | `~/.m2/repository/<group-path>/<artifact>/<version>/` | 查找 `-sources.jar` |
| Node.js (npm/pnpm/yarn) | `node_modules/<package>/` | 有 `.ts` / `.d.ts` 时可用；JS 库看 `src/` 或 `lib/` |
| Python (pip/uv) | `<venv>/lib/python*/site-packages/<package>/` | 或系统 site-packages |
| Go | `$GOPATH/pkg/mod/<module>@<version>/` | 只读模块缓存 |
| Rust | `~/.cargo/registry/src/<index>/<crate>-<version>/` | |
| .NET | NuGet 全局包目录（`~/.nuget/packages/` 或 `%USERPROFILE%\.nuget\packages\`） | |
| Swift | `~/Library/Developer/Xcode/DerivedData/*/SourcePackages/checkouts/` | SPM checkouts |
| Ruby | `$(gem env gemdir)/gems/<gem>-<version>/` | Bundler 或系统 gems |

### 检索顺序

1. **P0** — 先读项目自身代码
2. **P1** — 涉及第三方行为时，在上述典型路径中查找源码；路径不存在则搜索项目或全局缓存
3. **P2** — P1 源码不存在或不充分时，查询工具文档（如 Context7）；工具不可用则跳过
4. **P3** — P2 无覆盖或不可用时，搜索网络；无网络则跳过
5. **P4–P5** — 以上均不可用时，用记忆或训练知识降级，并标注未验证

## 训练知识豁免条件

### 可直接使用，无需外部验证（允许 P5）

- 语言标准语法（`for`、`if`、类型声明等稳定的基础语法）
- 通用算法和数据结构
- 纯推理任务（读代码 → 推断行为，不需要外部事实）
- 不随版本变化的框架通用模式（如"React 组件返回 JSX"）

### 必须外部验证（禁止 P5）

- API 签名、参数名、返回值类型
- 版本特定行为（"从 X.Y 版本开始..."、"在...中废弃"）
- 任何 API 或功能的废弃/移除状态
- 构建工具配置语法和选项（Gradle、Maven、webpack、Vite 等）
- 错误信息的含义和解决方案
- 安全相关断言（漏洞状态、安全默认值）
- 外部产品的定价、可用性或服务状态

## 与其他规则的关系

本规则是证据来源的操作化细则。如果项目中存在以下类型的规则，本规则与其互补而非冲突：

- **外部核验类规则**（如"优先使用官方文档验证"）— 本规则用具体层级表将该原则操作化。
- **反模式类规则**（如"禁止无证据断言"、"禁止信任过时记忆"）— 本规则定义每个层级什么算合格的证据。
- **持久上下文类规则**（如"当前代码状态覆盖记忆"）— P4 的定位与此一致：记忆补充而非替代实时证据。

当其他规则与本规则冲突时，以更严格的要求为准。
