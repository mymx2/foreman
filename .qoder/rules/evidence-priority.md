# 证据来源优先级

进行任何技术判断时，按以下证据层级执行。当高层级来源**可用且相关**时，优先使用高层级，禁止用低层级替代。某层级不可用时（工具缺失、无网络、目录不存在），跳过并降级到下一个可用层级，同时标注降级原因。

## 证据层级

| 层级 | 来源 | 适用场景 | Token 成本 |
|------|------|----------|------------|
| P0 | **工作区代码**（如文本搜索、符号解析、终端执行、数据库查询等） | 读取项目文件、运行构建和测试、查看数据库结构 | 低（本地读取） |
| P1 | **依赖源码文件**（如 IDEA-MCP、LSP 等） | 理解第三方库行为、API 内部实现、类型签名 | 低–中 |
| P2 | **文档查询工具**（如 Context7、DeepWiki 等） | 框架、库、构建工具的官方文档。工具不可用时跳过 | 中 |
| P3 | **网络检索**（如 WebSearch/WebFetch 等） | 最新资讯、P2 未覆盖的内容、错误信息、发布说明。无网络时跳过 | 中–高 |
| P3.5 | **浏览器工具**（如 Playwright MCP、内置 browser-use 等） | WebFetch 失败的 JS 渲染页面、反爬严格的站点、需要交互才能看到的内容。MCP 不可用时回退 CLI（如 `npx playwright`），见下文 | 高 |
| P4 | **结构化记忆**（如 SearchMemory 等记忆工具） | 用户偏好、项目模式、跨项目经验。工具不可用时跳过 | 低 |
| P5 | **训练知识** | 所有外部来源均不可用时的最终降级，或安全豁免主题（见下文） | 零 |

### 核心约束

- **P0–P1**：必须用工具读取实际文件。禁止凭记忆引用文件内容。
- **P2–P3**：涉及 API 签名、版本特定行为、废弃状态或构建工具配置时，**若工具可用**必须使用。
- **P4**：作为补充上下文。行动前应用更高可用层级验证事实。
- **P5**：所有外部来源不可用时允许作为最终降级。用于非平凡判断时，标注 `(inferred: 未联网验证)`。若属于下文「禁止 P5」的领域，即使无外部工具也应明确告知用户无法验证，而非自信给出结论。

### 降级策略

当某层级不可用时，不阻塞任务，直接降级到下一个可用层级：

| 不可用情况 | 处理方式 |
|------------|----------|
| P1 依赖源码目录不存在 | 跳到 P2/P3；P2/P3 不充分且需要源码级验证时，按临时 clone 规则 shallow clone |
| 文档查询工具不可用 | 跳到 P3 网络检索，或 P5 标注未验证 |
| WebFetch 返回空/不完整 | 跳到 P3.5 浏览器工具（Playwright MCP / browser-use 等），用真实浏览器渲染页面提取内容 |
| 浏览器 MCP 不可用 | 通过终端调用浏览器 CLI（如 `npx playwright`）截图或提取页面文本。CLI 也不可用时跳到 P4 记忆 + P5 标注未验证；JS 渲染 SPA 无任何浏览器工具时标注 `(JS-rendered: link inaccessible)` |
| 无网络访问 | 依赖 P0/P1 本地文件 + P4 记忆，P5 标注 `(offline)` |
| 记忆工具不可用 | 跳过 P4，直接 P5 |
| 所有外部来源均不可用 | P5 作为最终降级，但必须明确告知用户：哪些结论未经验证、风险范围、恢复验证的方法 |

## 链接内容获取策略

任务过程中遇到的 URL（GitHub 仓库、issue、PR、普通网页等）是 P0–P3 层级的重要证据来源。链接可能来自用户消息、代码注释、错误日志、配置文件、依赖文档、搜索结果等任何上下文。许多网站对自动化抓取有限制（反爬、登录墙、JS 渲染），网页抓取工具可能返回空内容或不完整内容。**不能因为一次获取失败就放弃该证据来源**——人类用户知道换个方式读，AI 必须被显式告知这些替代路径。

### GitHub 链接（仓库 / issue / PR / discussion / release）

GitHub 页面直接用网页抓取工具常被拦截或返回不完整内容。按以下优先级尝试：

| 优先级 | 能力层级 | 说明 |
|--------|----------|------|
| 1 | **结构化 API 访问** | 通过环境中的 CLI 或 MCP 工具调用 GitHub API，获取结构化数据（issue body、PR diff、release notes 等），不受网页反爬限制。检查当前环境是否有可用的 GitHub CLI 或 GitHub MCP 服务 |
| 2 | **底层 API 调用** | 当高层命令不满足时，直接调用 GitHub REST/GraphQL API 获取原始数据（如通过 CLI 的 `api` 子命令） |
| 3 | **网页抓取** | 用环境中的网页抓取工具获取页面 HTML/Markdown 内容。结构化 API 不可用时的降级 |
| 4 | **浏览器工具** | WebFetch 被反爬拦截或返回不完整时，用浏览器工具（Playwright MCP / 内置 browser-use 等）导航到页面提取内容。JS 渲染页面和反爬严格站点优先走此路径 |
| 5 | **本地克隆** | 将仓库 shallow clone 到项目已忽略的临时目录中读取源码文件（如 `git clone --depth 1 <repo_url> <ignored_dir>/<repo_name>`）。仅适用于源码读取，不适用于 issue/PR 内容 |

**关键规则：**

- **结构化 API 优先于网页抓取**：API 返回结构化数据，不受反爬限制。任何环境中有可用的 GitHub CLI 或 MCP 工具时，优先使用。
- **本地克隆仅用于源码**：issue/PR 的内容不在 git 仓库中，clone 无法获取。clone 只在需要读仓库文件时使用。
- **工具不可用时**：先确认当前环境是否有可用的 GitHub CLI 或 MCP 工具。没有时降级到下一优先级。

### 文档链接（库 / 框架 / 工具官方文档）

链接指向库或框架的官方文档时，P2 层的文档查询工具（如 Context7、DeepWiki 等提供结构化文档检索的服务）通常比直接网页抓取更优：返回结构化、版本匹配的内容，不受网页反爬和渲染问题影响。

| 优先级 | 能力层级 | 说明 |
|--------|----------|------|
| 1 | **文档查询工具（P2）** | 先用环境中的文档查询工具检索该库 / 框架的文档。覆盖到目标内容时直接使用，无需再抓取网页 |
| 2 | **网页抓取（P3）** | 文档查询工具未覆盖该库 / 版本，或内容不充分时，用网页抓取工具获取页面内容 |
| 3 | **网络搜索** | 抓取失败时，用搜索工具检索关键词，从结果摘要中提取 |
| 4 | **浏览器工具（P3.5）** | WebFetch 返回空内容或不完整时（常见于 JS 渲染的文档站如 SPA 文档），用浏览器工具导航到页面提取渲染后的完整内容 |

**判断规则**：链接指向的是知名库 / 框架的文档站（如 React、Next.js、Spring、Gradle 等）时，优先走 P2 而非直接抓取。个人博客、新闻报道、论坛帖子等非文档页面，直接走网页抓取。

### 其他网页链接

| 优先级 | 能力层级 | 说明 |
|--------|----------|------|
| 1 | **网页抓取** | 用环境中的网页抓取工具获取页面 HTML/Markdown 内容。大多数网页可用，优先尝试 |
| 2 | **网络搜索** | 抓取失败时，用环境中的搜索工具检索 URL 中的关键词，从搜索结果摘要中提取信息 |
| 3 | **浏览器工具** | WebFetch 被拦截或返回不完整时，用浏览器工具（Playwright MCP / 内置 browser-use 等）导航到页面提取内容。JS 渲染页面优先走此路径 |
| 4 | **告知用户** | 所有方法均失败时，明确告知用户无法获取该页面内容，请求用户提供文本摘要或截图 |

### 通用规则

- **不静默放弃，也不无限重试**：链接获取失败时，按降级链路尝试，同一层级最多尝试 2 次。全部失败后标注 `(link inaccessible)` 继续任务，不阻塞也不反复重试。在最终输出中列出未能获取的链接及原因。
- **禁止用 P5 替代链接内容**：任务中遇到的链接是 P0–P3 层级的实时证据，获取失败不能直接用训练知识替代，必须标注 `(link inaccessible)` 并说明风险。
- **多链接批量处理**：同一任务涉及多个链接时，逐个尝试获取，汇总报告每个链接的获取状态（成功 / 降级 / 失败）。
- **认证检测**：涉及 GitHub CLI 或 MCP 工具时，首次使用前检查认证状态（如 `gh auth status`）。未认证时提示用户完成认证，同时降级到下一优先级方法继续任务。
- **临时 clone 目录**：本地克隆的仓库源码存放于此，作为 P1 事实来源。需要克隆任何仓库到本地读取、对比或研究时，先检查是否已存在克隆目录，有则直接用。不存在时读项目 `.gitignore`，选一个已被忽略的根级目录名（如 `__source-clones`、`source-clones.tmp` 等，具体名称取决于已有的忽略规则）创建。禁止新增 `.gitignore` 条目，禁止使用未被忽略的路径。

### 浏览器 MCP → CLI 回退

当浏览器 MCP 工具（Playwright MCP、browser-use 等）不可用时，不应直接放弃浏览器能力。通过终端调用浏览器 CLI（如 `npx playwright`）作为降级手段：截图后 Read 图片，或执行脚本提取渲染后的页面文本。网络可用时 `npx` 会自动拉包执行，无需预装。

## 依赖源码目录映射

需要 P1 检索时，按生态系统在本地查找源码。以下为各生态系统的**典型路径**，具体路径因项目配置而异——若典型路径不存在，在项目目录或全局缓存中搜索对应的源码目录：

| 生态系统 | 典型本地源码路径 | 说明 |
|----------|-------------|------|
| Gradle (Kotlin/Java) | `<project>/.gradle/gradle_module/<group>/<artifact>/<version>/` | 需项目配置 `unzipSourceJars` 任务；未配置时回退到 `~/.gradle/caches/` 下查找 `-sources.jar` |
| Maven (Java) | `~/.m2/repository/<group-path>/<artifact>/<version>/` | 查找 `-sources.jar` |
| Node.js (npm/pnpm/yarn) | `node_modules/<package>/` | 有 `.ts` / `.d.ts` 时可用；JS 库看 `src/` 或 `lib/` |
| Python (pip/uv) | `<venv>/lib/python*/site-packages/<package>/` | 或系统 site-packages |
| Go | `$GOPATH/pkg/mod/<module>@<version>/` | 只读模块缓存 |
| Rust | `~/.cargo/registry/src/<index>/<crate>-<version>/` | |
| .NET | NuGet 全局包目录（`~/.nuget/packages/` 或 `%USERPROFILE%\.nuget\packages\`） | |
| Swift | `~/Library/Developer/Xcode/DerivedData/*/SourcePackages/checkouts/` | SPM checkouts |
| Ruby | `$(gem env gemdir)/gems/<gem>-<version>/` | Bundler 或系统 gems |

### 检索顺序

1. **P0** — 先读项目自身代码
2. **P1** — 涉及第三方行为时，在上述典型路径中查找源码；路径不存在时跳到 P2/P3；P2/P3 不充分且需要源码级验证时，按临时 clone 规则 shallow clone，不询问
3. **P2** — P1 源码不存在或不充分时，查询文档工具（如 Context7、DeepWiki 等）；工具不可用则跳过
4. **P3** — P2 无覆盖或不可用时，搜索网络；无网络则跳过
5. **P3.5** — P3 获取不完整时（JS 渲染页面、反爬拦截），用浏览器工具渲染页面提取内容；MCP 不可用时回退 CLI；CLI 也不可用则跳过
6. **P4–P5** — 以上均不可用时，用记忆或训练知识降级，并标注未验证

## 训练知识豁免条件

### 可直接使用，无需外部验证（允许 P5）

- 语言标准语法（`for`、`if`、类型声明等稳定的基础语法）
- 通用算法和数据结构
- 纯推理任务（读代码 → 推断行为，不需要外部事实）
- 不随版本变化的框架通用模式（如"React 组件返回 JSX"）

### 必须外部验证（禁止 P5）

- API 签名、参数名、返回值类型
- 版本特定行为（"从 X.Y 版本开始..."、"在...中废弃"）
- 任何 API 或功能的废弃/移除状态
- 构建工具配置语法和选项（Gradle、Maven、webpack、Vite 等）
- 错误信息的含义和解决方案
- 安全相关断言（漏洞状态、安全默认值）
- 外部产品的定价、可用性或服务状态

## 与其他规则的关系

本规则是证据来源的操作化细则。如果项目中存在以下类型的规则，本规则与其互补而非冲突：

- **外部核验类规则**（如"优先使用官方文档验证"）— 本规则用具体层级表将该原则操作化。
- **反模式类规则**（如"禁止无证据断言"、"禁止信任过时记忆"）— 本规则定义每个层级什么算合格的证据。
- **持久上下文类规则**（如"当前代码状态覆盖记忆"）— P4 的定位与此一致：记忆补充而非替代实时证据。

当其他规则与本规则冲突时，以更严格的要求为准。
