# irq-balance 示例 Prompt

以下是触发 irq-balance skill 的典型用户 prompt 示例，覆盖各类使用场景。

## 初步诊断

```
帮我分析一下当前服务器的中断负载分布情况
```

```
用 mpstat 看了下，CPU0 的 %soft 占比明显高于其他 CPU，帮我排查原因
```

```
/proc/softirqs 里 NET_RX 在某个 CPU 上特别高，帮我分析
```

## 软中断不均衡

```
系统软中断集中在 CPU3，其他 CPU 几乎没有，帮我排查
```

```
NET_RX 软中断全落在 CPU0，怎么分散到其他 CPU
```

```
ksoftirqd/2 这个线程 CPU 占用持续 80% 以上，帮我定位原因
```

```
软中断分布不均导致网络延迟抖动，帮我用 perf-prof 分析
```

## 硬中断不均衡

```
网卡中断全集中在 CPU0，其他 CPU 没有收到硬中断，怎么处理
```

```
帮我检查 eth0 的 IRQ 亲和性配置是否合理
```

```
/proc/interrupts 里 virtio0-input.3 的计数是其他队列的 4 倍，帮我分析
```

## 网络收包不均匀

```
网络收包性能不均衡，部分 CPU 跑满但其他 CPU 空闲，帮我排查
```

```
帮我分析 eth0 的 RSS 哈希分布，判断网卡队列是否均衡
```

```
服务器有 8 个网卡队列，但流量全走队列 3，帮我排查中断分布
```

```
开启 RPS 后 NET_RX 是否均衡了，帮我用 perf-prof 验证效果
```

## 中断处理延迟

```
软中断处理延迟 p99 超过 1ms，帮我用 multi-trace 分析原因
```

```
帮我统计各 CPU 上软中断处理的 p50/p95/p99 延迟
```

```
NET_RX 软中断偶发高延迟，帮我找出大延迟时的调用路径
```

## 热点定位

```
中断负载最高的 CPU 上有什么热点函数，帮我生成火焰图
```

```
帮我采样 ksoftirqd 线程的调用栈，看看它在做什么
```

```
net_rx_action 在火焰图里占比很高，帮我解读
```

## 综合排查

```
系统网络吞吐达不到预期，部分 CPU 已满载，帮我做中断均衡性排查
```

```
帮我做一次完整的中断不均衡排查，从诊断到根因确认
```

```
irqbalance 没有运行，现在中断分布不均，帮我分析并给出优化建议
```
